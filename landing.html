What Auth Mesh Is

Auth Mesh is the identity and permission layer for your entire platform. One system that handles authentication, fine-grained authorization, and organizational structure.

Zero-Config Permissions

Permissions from membership,
not per-user grants.

Users get permissions by being members of an org. The org inherits permissions from its parent through Zanzibar relationships. No per-user grants. No cron jobs to sync permissions. No callbacks to handle new signups. Membership is the permission.

How It Works

Three minutes to working auth.

The setup CLI creates your service org, registers permissions, configures OAuth, and provisions your user structure. Every step is idempotent. Run it once. Run it again. Same result.

01

Configure

Define your permissions, OAuth client, and login branding in three JSON files. Copy from examples, edit for your service.

02

Run Setup

One command. The CLI registers your service, creates the org hierarchy, configures hosted login, and verifies everything.

03

Integrate

Point your frontend at the hosted login URL. Add the SDK. Users sign up, log in, and get permissions automatically.

# That's it. Really. cp config/permissions.json.example config/permissions.json cp config/oauth-client.json.example config/oauth-client.json cp config/hosted-login.json.example config/hosted-login.json # Edit each file for your service, then: ./setup run # Output: # [01/05] Registering service permissions... done # [02/05] Registering OAuth client... done # [03/05] Configuring hosted login... done # [04/05] Setting up end-users org... done # [05/05] Verifying setup... all checks passed # # Your hosted login: https://auth.service.ab0t.com/login/my-service-users # Credentials saved to: credentials/

Grant another user scoped, time-limited ability to act on your behalf. Perfect for vacation coverage, support access, admin assistance. The actor gets a delegated JWT with both identities. Can only delegate permissions you have. Revoke instantly.

Scoped TTL Delegated JWT Audit Trail
# Grant delegation POST /delegation/grant {"actor_id": "user_bob", "scope": ["approvals.write"], "expires_in_hours": 48} # Bob acts as Alice POST /auth/delegate {"target_user_id": "user_alice"} # Returns JWT with both identities

Each org can configure its own SAML IdP. A hospital system can have Epic for clinical orgs and Okta for admin orgs. JIT provisioning auto-creates users from SAML assertions and adds them to the org. Integrates with the OAuth PKCE flow — SSO feeds into the standard token exchange.

Per-Org IdP SAML 2.0 JIT Provisioning OAuth Integration
